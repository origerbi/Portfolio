Titanic survival prediction¶

Ori Gerbi¶

Kaggle Link: https://www.kaggle.com/origerbi96¶

At this Kaggle competition "Titanic - Machine Learning from Disaster", I wish to categorize the titanic passengers to survivors and victims using Machine Learning. In this notebook, I will:

  • Investigate the dataset in order to find connections between passengers and their impact on the chances of survival.
  • View possible relations through barplots and charts.
  • Perform classification on the data.

Tools:¶

  • Logistic Regression - SGD Classifier, which is required to do the binary classification (Survived - Died).
  • Pandas & Seaborn, which allowed me to organize the data and display it.

def setMissingValuesAsMeanValues(df,column):                  # Replacing the missing values with the mean.
    df[column].fillna(df[column].mean(),inplace = True)

def setMissingValuesAsMostOccurrens(df,column):                # Replacing the missing values with the most occurred number of times.
    df[column].fillna(df[column].mode()[0],inplace = True)

Data Slicing¶

We can not check the performence of the model on the same dataset that the model was trained on.

This will result in wrong estimation for the model generalization capabilities.

In order to check our prediction and fine-tune the model parameters, we need to slice the dataset into 2 groups:

  • train
  • validation

We will train on the train data and check the performance on the validation data.

We will slice the dataset with Scikit-learn train_test_split.

First, let's split the data to features X and target t.

Classification¶

We can use 2 ways to classify the data based on logistic regression:

  • The Optimizer - SGDClassifier (When using log loss)
  • The Estimator - LogisticRegression

We will use the SGDClassifier

First, we want to check how we should split our dataset so it will give us the best result.

In [56]:
def scatter_points_on_graph(graphPoints):
    for k, v in graphPoints.items():
        color = "Red" if "Train" in k else "Blue"
        sns.scatterplot(data = graphPoints, x = v.keys(), y = v.values())
        plt.title(k)
        plt.show()
        
# plot the score by split and the loss by split
def display_score_loss_by_split(x, t):
    graph_points = { "Train CE" : {}, "Validation CE": {},"Train Accuracy": {},  "Validation Accuracy": {} }
    for size in range(10, 100, 10):
        x_train, x_validation, t_train, t_validation = model_selection.train_test_split(x, t, test_size = size / 100, random_state=42)
        SGDClassifier = pipeline.make_pipeline(preprocessing.StandardScaler(), linear_model.SGDClassifier(loss = "log", alpha = 0, learning_rate = "constant", eta0 = 0.01)).fit(x_train, t_train)
        y_train_prob = SGDClassifier.predict_proba(x_train)
        y_validation_prob = SGDClassifier.predict_proba(x_validation)
        
        graph_points["Train CE"][size / 100] = metrics.log_loss(t_train, y_train_prob)
        graph_points["Validation CE"][size / 100] = metrics.log_loss(t_validation, y_validation_prob)
        graph_points["Train Accuracy"][size / 100] = SGDClassifier.score(x_train, t_train)
        graph_points["Validation Accuracy"][size / 100] = SGDClassifier.score(x_validation, t_validation)
    scatter_points_on_graph(graph_points)

display_score_loss_by_split(X, t)

The score function in Scikit-learn SGDClassifier is the mean accuracy for each label.

The accuracy is defined as the ratio between the number of correct predictions and all the predictions (it means, how correct the model is).

The accuracy values are in range [0,1].

The higher the score, the better the model.

We can see that when the validation data size is small, its loss is small. One explanation to this is that for a small group of samples, it is easier to match a linear hypothesis.

So, let's give the validation group 20% of the dataset, it is about the right point where the validation loss is equal to the train loss.

Summary¶

In this contest, I attempted to predict who of the Titanic passengers survived.

I decided to remove from the dataset some features like: passenger's name and cabin number. Since it would have seemed irrelevant or lacked a lot of information (as in the cabin number, which made it difficult to make a good classification).

I used data analysis shown by charts in order to show possible relations between passengers' categories (Sex, Age, Pclass and more...).

I saw that the majority who survived were women and children, (probably people gave them priority to lifeboats).

I also saw that most of those who survived were in first class (probably those who pay more are more likely to survive).

After that, I ran several models and each one was distributed differently to see how to divide the dataset to get the least error. I came to the conclusion that it is best to give 20% to validation and the rest to train.

Finally, I predicted the test dataset and submitted the file to the competition.
